Tema 1: Elementos de página web

  1. Origen de la web

    2. La web fue propuesta por Tim Berners-Lee en 1989 mientras trabajaba en el CERN. En 1990, desarrolló HTML y el sistema URI para identificar recursos. En 1991, creó el primer navegador y servidor web, permitiendo el acceso a páginas por primera vez. La popularidad creció con el navegador Mosaic en 1993, y en 1994 se fundó el W3C para estandarizar la web. A finales de los 90, la web se comercializó, transformando la forma en que las personas interactúan e intercambian información.

    1. Web 2.0

      2. La Web 2.0 es la evolución de la web hacia una plataforma más interactiva y colaborativa. La Web 2.0 permite a los usuarios crear, compartir y colaborar en contenido. Sus características principales incluyen redes sociales, contenido generado por los usuarios, herramientas colaborativas como wikis y aplicaciones web. En resumen, la Web 2.0 hace que la experiencia en línea sea más dinámica y participativa.

    2. Web 3.0

      3. La Web 3.0, o "Web descentralizada", es la próxima fase de la web que busca empoderar a los usuarios mediante tecnologías como blockchain. Sus características clave son:

      • Descentralización: Menos control por grandes empresas.
      • Propiedad de los datos: Los usuarios controlan sus propios datos.
      • Inteligencia artificial: Mejora la personalización y la búsqueda.
      • Interoperabilidad: Conexión fluida entre plataformas.
      • Aplicaciones descentralizadas (dApps): Ofrecen más transparencia y seguridad.

    3. La W3C y los estándares

      4. La W3C (World Wide Web Consortium) es una organización que desarrolla estándares para asegurar el crecimiento y la interoperabilidad de la web. Fundada por Tim Berners-Lee, sus funciones principales incluyen:

      • Desarrollo de estándares: Especificaciones técnicas como HTML y CSS.
      • Accesibilidad: Promoción de pautas para que la web sea accesible a todos.
      • Interoperabilidad: Facilita la compatibilidad entre tecnologías.
      • Innovación: Fomenta nuevas tecnologías para mantener la web actualizada.

      Los estándares Web W3C son protocolos, pautas y tecnologías inter-operativas e internacionales creadas con la finalidad de guiar la Web a un mejor rendimiento y mejorar la funcionalidad operativa para los navegadores y usuarios.

  2. Lenguajes de etiquetas

    1. SGML (Standard Generalized Markup Language)

      2. Es un estándar para definir lenguajes de marcado. Permite la creación de documentos estructurados. SGML fue la base de varios lenguajes de marcado, incluyendo HTML y XML.

    2. XML (eXtensible Markup Language)

      3. Es un lenguaje de marcado que permite crear etiquetas personalizadas. Se utiliza para almacenar y transportar datos de manera estructurada. Es flexible y se usa en aplicaciones diversas, desde servicios web hasta configuraciones.

    3. HTML (HyperText Markup Language)

      4. Es un lenguaje marcado de hipertexto, sirve para describir la estructura básica de una página y organizar la forma en que se mostrará su contenido, además de que HTML permite incluir enlaces hacia otras páginas o documentos.

    4. XHTML (eXtensible HyperText Markup Language)

      5. Lenguaje de marcado de hipertexto extensible). Es un cruce entre el lenguaje de programación HTML y XML. XHTML es casi lo mismo que HTML, pero es más rígido que HTML. XHTML es una aplicación XML especificada en HTML.

    5. HTML5

      6. Es la última versión de HyperText Markup Language (Lenguaje de Marcado de Hipertexto). Combina tres tecnologías web: HTML, para construir la estructura de la página; CSS, para mejorar la presentación; y JavaScript, para agregar funcionalidad a los elementos HTML.

  3. Otros lenguajes relacionados al desarrollo web

    1. CSS/CSS3

      2. CSS (Cascading Style Sheets) es un lenguaje usado para describir la presentación de documentos HTML. CSS3 es la última versión que incluye nuevas características como animaciones, flexbox y gradientes.

    2. JavaScript

      3. Un lenguaje de programación que permite implementar interactividad en las páginas web. Se utiliza para crear efectos dinámicos, validar formularios y manejar eventos del usuario.

    3. Ajax (Asynchronous JavaScript and XML)

      4. Técnica que permite la actualización asíncrona de contenido en una página web sin recargarla. Facilita la interacción fluida con el servidor, permitiendo cargar datos en segundo plano y mejorar la experiencia del usuario.

    4. Tecnologías y lenguajes del lado del servidor

      5. Las tecnologías y lenguajes del lado del servidor son herramientas utilizadas para desarrollar la lógica de aplicaciones que se ejecutan en el servidor. Su función principal es procesar las solicitudes del cliente, gestionar bases de datos y generar respuestas. Algunos ejemplos son:

      Node.js, Python, Java, Ruby, PHP, C#, Go, Rust.

      Frameworks como Django, Flask, Spring, Ruby on Rails y ASP.NET.

    5. Navegadores

      6. Los navegadores son aplicaciones que permiten a los usuarios acceder y visualizar contenido en la web. Ejemplos populares incluyen:

      • Google Chrome
      • Mozilla Firefox
      • Microsoft Edge
      • Safari

      Las características de los navegadores incluyen motores de renderizado, soporte para estándares web, y herramientas para desarrolladores.

    6. Motores renderizados

      7. Los motores de renderizado son componentes que interpretan y muestran el contenido web en los navegadores. Se encargan de convertir el código HTML, CSS y JavaScript en la interfaz visual que los usuarios ven. Ejemplos incluyen:

      • Blink (Chrome y Edge)
      • Gecko (Firefox)
      • WebKit (Safari)

      Cada motor tiene sus propias optimizaciones y características, lo que puede afectar la forma en que se visualiza el contenido en diferentes navegadores.

  4. Herramientas de desarrollo

    1. Software WYSIWYG

      2. El software WYSIWYG (What You See Is What You Get) permite a los usuarios crear y editar contenido de forma visual, mostrando exactamente cómo se verá en el navegador sin necesidad de escribir código. Ejemplo:

      • Adobe Dreamweaver: Permite la edición tanto visual como de código.
      • WordPress: Ofrece editores visuales como Elementor y Gutenberg.
      • Wix y Squarespace: Plataformas que permiten construir sitios web de manera sencilla.

    2. Otras alternativas

      3. Además del software WYSIWYG, existen otras herramientas y plataformas que ofrecen diferentes enfoques para la creación y gestión de contenido, especialmente en el diseño y desarrollo web.

      1. Figma

        2. Es una herramienta de diseño colaborativo basada en la web que permite a los usuarios crear interfaces de usuario, prototipos y diseños gráficos. Es ampliamente utilizada por diseñadores y equipos de desarrollo para trabajar en proyectos de diseño de productos digitales.

      2. InVisión

        3. Es una herramienta de diseño de interfaz de usuario (UI) y experiencia de usuario (UX) utilizada para la creación de prototipos interactivos y de alta fidelidad. Esta plataforma permite a los diseñadores colaborar en tiempo real, facilitando la comunicación y el intercambio de ideas en todas las etapas del proceso de diseño.

      3. Zeplin

        4. Es una herramienta que conecta diseñadores y desarrolladores, generando especificaciones y activos de diseño para facilitar la entrega y mejorar la colaboración.

      4. GitHub

        5. Es un servicio basado en la nube que aloja un sistema de control de versiones (VCS) llamado Git. Éste permite a los desarrolladores colaborar y realizar cambios en proyectos compartidos, a la vez que mantienen un seguimiento detallado de su progreso.

    3. Inspección de código desde el navegador.

      4. La inspección de código desde el navegador se refiere al uso de herramientas de desarrollo integradas en navegadores web que permiten a los usuarios ver y modificar el código fuente de una página web en tiempo real. Esta funcionalidad es especialmente útil para desarrolladores y diseñadores web.

    1. Tipos de páginas Web

      • Sitios Web Estáticos:

        • Son páginas web cuyo contenido es fijo y no cambia a menos que se modifique manualmente el código. Se componen principalmente de archivos HTML y CSS, y cada página es un documento independiente.

      • Sitios web dinámicos:

        • Son páginas que generan contenido en tiempo real y pueden cambiar según la interacción del usuario, datos de bases de datos o configuraciones específicas. Utilizan tecnologías del lado del servidor como PHP, Python, Ruby o Node.js, así como bases de datos como MySQL o MongoDB.

      • E-commerce

        • son sitios web diseñados específicamente para la compra y venta de productos o servicios a través de Internet. Facilitan todo el proceso de transacción, desde la navegación hasta el pago y el envío.

      • Blogs:

        • son plataformas en línea que permiten a los usuarios crear, publicar y gestionar contenido en formato de entradas o artículos. Estos sitios pueden ser personales, profesionales o temáticos y suelen facilitar la interacción con los lectores a través de comentarios y comparticiones.

      • Foros

        • son plataformas en línea que permiten a los usuarios participar en discusiones y compartir información sobre una amplia variedad de temas. Funcionan como comunidades virtuales donde los miembros pueden crear publicaciones, comentar y responder a otros usuarios.

      • Los portales web

        • son sitios que actúan como puntos de entrada a una variedad de recursos y servicios en línea. Estos portales ofrecen un acceso centralizado a información, herramientas y enlaces a otros sitios, facilitando la navegación y el uso de Internet.

    2. Puntos básicos en la elaboración de una página Web.

      1. Planificación

        2. Definir el propósito y los objetivos de la página. Identificar al público objetivo y entender sus necesidades para crear contenido relevante.

      2. Estructura y Diseño

        3. Crear un esquema o wireframe para organizar el contenido. Elegir un diseño visual coherente que refleje la identidad de la marca, incluyendo colores, tipografía y elementos gráficos.

      3. Desarrollo del Contenido

        4. Redactar textos claros y concisos, complementándolos con imágenes y multimedia de alta calidad. Asegurarse de que el contenido sea accesible y atractivo para los usuarios.

      4. Elección de Tecnologías

        5. Decidir qué lenguajes de programación utilizar (HTML, CSS, JavaScript) y considerar el uso de lenguajes del lado del servidor como PHP o Python. Evaluar la posibilidad de usar un CMS como WordPress.

      5. Optimización para Dispositivos Móviles

        6. Asegurarse de que la página sea responsiva, adaptándose a diferentes tamaños de pantalla para ofrecer una experiencia de usuario fluida en todos los dispositivos.

      6. SEO (Optimización para Motores de Búsqueda)

        7. Implementar prácticas de SEO, como el uso de etiquetas HTML adecuadas y contenido optimizado con palabras clave relevantes, para mejorar la visibilidad en los motores de búsqueda.

      7. Pruebas y Validación

        8. Realizar pruebas en diferentes navegadores y dispositivos para garantizar que todo funcione correctamente. Validar el código para detectar errores y mejorar la accesibilidad.

      8. Publicación y Mantenimiento

        9. Elegir un proveedor de alojamiento web y un nombre de dominio para lanzar la página. Planificar actualizaciones periódicas y resolver problemas técnicos de manera oportuna.

      9. Análisis y Mejoras

        10. Utilizar herramientas de análisis para rastrear el tráfico y el comportamiento del usuario. Recoger feedback y ajustar la página según sea necesario para mejorar la experiencia del visitante.
